# 前程无忧 OCR 投递工作流

当前版本新增前程无忧基础工作流，不替换 Boss、猎聘、智联入口。四个平台的审查流程保持一致：先用同一个简历画像匹配当前岗位详情，再允许执行平台投递动作。

## 统一审查流程

截图 -> OCR -> 抽取当前岗位详情 -> DeepSeek 使用复用简历画像判断 -> Mimo 根据平台 workflow 决策 -> 投递动作门禁 -> 检查结果 -> 返回列表继续。

- 简历全文只用于首次生成 `resume_profile`。
- 每个岗位只发送 `resume_profile` + 当前岗位详情文本。
- DeepSeek 返回 `fit=apply` 且分数达到阈值，才允许点击申请/投递按钮。
- 快速模式后续再加：届时可跳过 DeepSeek，只用标题/关键词做粗匹配；当前基础流程不启用快速模式。

## 前程无忧页面状态

1. 弹窗或浮层
   - 信号：`关闭`、`X`、`我知道了`、`求职助手`、`订阅`、`APP`、小程序提示
   - 动作：只关闭明显营销/订阅/APP 浮层，不处理登录或验证。

2. 搜索页
   - 信号：`搜索职位`、`搜索职位、公司`、`关键词`、`地区`、`搜索`
   - 动作：输入目标关键词并搜索。

3. 搜索结果页
   - 信号：`综合排序`、`活跃职位优先`、`最新优先`、岗位卡、公司、薪资、标签、`去聊聊`、`投递`
   - 动作：可以直接点同一岗位卡上的 `投递`，也可以点岗位标题进入详情页。
   - 约束：列表页直接 `投递` 前也必须先过 DeepSeek 审查；如果列表卡片信息不足，DeepSeek 应返回 `need_more_detail`，让 Mimo 点标题进详情页。

4. 岗位详情页
   - 信号：`职位描述`、`岗位职责`、`任职要求`、`职位招聘官`、`公司信息`、`立即投递`
   - 动作：抽取当前详情页主体文本给 DeepSeek 审查。

5. 投递动作
   - 信号：列表页 `投递`，详情页 `立即投递`，以及确认弹窗里的 `确定投递/确认投递`
   - 动作：这些都算最终投递动作，必须通过 DeepSeek 门禁。
   - 默认不点 `去聊聊`，它是聊天动作，不是前程无忧默认投递路径。

6. 成功或已投递
   - 信号：`投递成功，前往应届生求职App随时查看更多职位`、`竞争力分析`、`立即查看`、`没有职位不要怕`、`回首页再看看`、`已申请`、`已投递`
   - 动作：记为成功，然后关闭当前浏览器标签页，使上一标签的搜索结果页重新激活，再继续下一岗位。
   - 关闭方式：优先点击当前 Chrome 标签页顶部 `X`，或使用 `Ctrl+W`。不要点击左边/上一页的页面内容来返回。

## 禁止动作

- 不自动处理登录、扫码、短信、验证码、人机验证、滑块。
- 不默认点击 `去聊聊`。
- 不上传本地简历，不切换附件简历，不修改简历。
- 看到 `完善简历`、`请选择简历`、`上传简历`、`附件简历`、`简历不完整` 时默认 `need_user`。
- 未通过 DeepSeek 审查时不点击 `投递/立即投递/确定投递/确认投递/投递简历`。
