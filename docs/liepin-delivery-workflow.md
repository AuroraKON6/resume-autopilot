# 猎聘 OCR 投递工作流

依据 `E:\学习\猎聘操作指南.docx` 设计。该流程是新增猎聘专属策略，不替换 Boss、51job、智联流程；四个平台仍可通过 `--platform boss|51job|zhilian|liepin` 分别运行。

## 目标

截图 -> OCR -> Mimo 视觉理解 -> DeepSeek 简历画像匹配 -> 范围点击执行 -> 成功/沟通状态检查 -> 回到列表继续。

## 页面状态

1. 首页或列表页弹窗
   - 信号：`618特惠季`、`立即抢购`、会员广告、`X/关闭`
   - 动作：只点可见关闭按钮，关闭后继续搜索或列表。

2. 搜索页
   - 信号：`搜索职位、公司`、`搜索职位`、`搜索`
   - 动作：输入目标关键词并搜索。

3. 搜索结果页
   - 信号：`综合`、`最新`、`招聘者活跃`、`应届生`、`实习生`、HR 信息
   - 动作：点击岗位卡进入详情页，不在列表摘要里直接投递。

4. 快速模式卡片筛选
   - 信号：职位卡片左上标题、薪资、城市、`应届`、`经验不限`、`大专`、`本科`、HR/招聘者区域、`聊一聊`。
   - 动作：启用 `--fast-mode` 时，先把可见 OCR 卡片文本交给配置的 API AI 做保守筛选；Hermes/本地代码只负责执行、截图、OCR、门禁和返回。
   - 右上角广告、二维码、App 下载提示不参与判断，可以忽略。
   - 先看中间经验/学历标签是否符合用户画像；`大专` 只是截图里的例子，实际要让 API AI 比较 `大专/本科/硕士/博士/学历不限/及以上` 等学历信号和用户画像是否匹配。
   - 再看左上标题/方向是否和用户目标相似，例如 Python/爬虫/数据分析等关键词。
   - 如果卡片匹配但 `聊一聊` 未显示，先 hover 到同一张卡片或招聘者区域，让按钮出现；按钮出现后再点击 `聊一聊`，并设置 `is_final_submit=true`。

5. 详情页审查
   - 信号：`职位介绍`、`岗位职责`、`任职要求`、`公司信息`、`聊一聊`、`投简历`
   - 动作：抽取当前详情页主体文本给 DeepSeek 判断。
   - 只有 `fit=apply` 且分数达到阈值时，Mimo 才能点击 `聊一聊` 或 `投简历`。

6. 校招项目详情页
   - 信号：`校招介绍`、`校招职位`、`职位筛选`、`部门/机构`、`城市`、`职位`、`到底了`。
   - 动作：这是当前打开的详情/项目标签页，下面的岗位卡仍按 OCR + API AI 审查。
   - 投递或沟通成功后，点击当前浏览器标签页顶部的 `X`，或用 `Ctrl+W` 关闭当前标签页，让上一搜索/列表标签页重新激活。
   - 不要点击左边或上一页的页面内容来返回。

7. 沟通/已投递状态
   - 信号：`我的沟通`、`继续聊`、已发送的打招呼内容、`投递成功`、`已投递`、`已沟通`
   - 动作：记为一次成功投递，然后关闭当前详情/项目标签页回到上一搜索结果页。

## 禁止动作

- 不自动处理登录、扫码、短信、验证码、人机验证、滑块。
- 不默认点击 `发简历`、`交换手机号`、`交换微信号`。
- 不上传、选择、修改简历文件；简历只用于生成一次可复用画像，并作为岗位匹配上下文。
- 不对未通过 DeepSeek 匹配审查的岗位点击 `聊一聊` 或 `投简历`。

## Mimo 决策要点

- 点击动作优先返回按钮范围 `box: {x1,y1,x2,y2}`，程序会在安全内边距内随机取点，不固定点击中心。
- 发行版的快速模式岗位判断必须来自配置的 API AI；本地规则只允许做安全兜底，例如验证码拦截、最终投递门禁、点击范围随机化。
- 猎聘快速模式允许 `hover` 动作，用于把鼠标移动到匹配岗位卡片或招聘者区域，显示隐藏的 `聊一聊` 按钮；`hover` 不算投递动作。
- 猎聘 `聊一聊` 和 `投简历` 都算最终投递/沟通动作，必须设置 `is_final_submit=true`。
- 遇到 `继续聊` 或 `我的沟通` 且能看到已发送打招呼内容时，不继续点聊天工具，直接判定已完成并关闭当前标签页。关闭方式优先是当前 Chrome 标签页顶部 `X` 或 `Ctrl+W`，不要点击旧页面内容。

## DeepSeek 匹配要点

- 简历全文只在首次生成 `resume_profile` 时发送一次，后续每个岗位只发送复用画像 + 当前岗位详情。
- 猎聘详情页抽取范围包含标题、标签、`职位介绍`、`岗位职责/任职要求`、`公司信息`。
- 用户目标包含实习、应届、校招时，必须看到对应信号才允许投递。
